### Cercus 实验控制框架使用指南

**Cercus** 是一个面向高时间精度生物行为学与神经科学实验的多进程、闭环刺激控制框架。该系统采用主从架构（Master-Worker），实现了 UI 调度、视觉渲染、硬件遥测与数据持久化的物理级解耦。

#### 1. 架构概览

系统强制执行严格的单向数据流与功能隔离：

- **主控中枢 (`dashboard.py`)**：非阻塞 GUI，负责参数装配、进程生命周期管理与低频状态监控（数字孪生）。
- **纯逻辑计算核 (`paradigm.py`)**：无渲染器依赖的数学模型层。基于时间差与硬件反馈，输出标准化渲染指令流与遥测状态。
- **无状态渲染引擎 (`core_render.py`)**：PsychoPy 绘图容器。仅执行来自计算核的基础几何绘制指令，支持渲染器即插即用。
- **异步硬件驱动 (`core_hardware.py`)**：独立线程守护的串口通信层，处理极高频外部传感器数据采集与触发信号下发。
- **双轨数据记录 (`core_logger.py`)**：分离极高频运动学遥测数据（Kinematics Log）与低频实验状态跃迁数据（Event Log）。

#### 2. 核心功能与执行模式

- **双轨执行模式 (Execution Mode)**：
    - **Auto (自动模式)**：行为学标准流程。基于设定的 ITI/ISI 随机区间，全自动连续执行整个 Session。
    - **Manual (手动模式)**：电生理标准流程。在 ITI 结束后，渲染进程安全挂起（保持基线与硬件心跳），等待外部输入（按 `Space` 键）以精准触发单次试次。
- **数字孪生监控**：主控 UI 包含实时微缩监视器，将 3840x1080 的双屏刺激状态等比映射至控制端，实现全闭环监控。
- **硬件降级与虚拟流**：无硬件接入时，串口可指定为 `mock` 注入虚拟数据流，保障调试链路贯通。

#### 3. 内置实验范式 (Paradigms)

系统目前已内置两套标准范式矩阵，均可通过 UI 面板直接调用与配置：

1. **多模态逼近 (Looming Paradigm)**：
    - 包含纯视觉基线 (`Baseline Visual`)、纯风场基线 (`Baseline Wind`)。
    - 包含 7 组精准标定的视触觉多感觉融合刺激 (`Looming + Wind`)，提供从 TTC -373ms 到 +200ms 的梯度风场硬件触发逻辑。
2. **经典视觉逼近 (ClassicLooming Paradigm)**：
    - 纯视觉参数化模型。支持在 UI 动态配置碰撞比 `l/v Ratio (ms)`、起始视角 (`Initial Degree`) 与终止视角 (`Final Degree`)。
    - 支持指定呈现逻辑：`Random L/R`（随机左右）、`Always Left`（恒定左侧）、`Always Right`（恒定右侧）。

#### 4. 环境安装

建议在隔离的虚拟环境（如 Conda）中运行本系统。核心依赖包括 UI、视觉渲染与硬件通信模块：

```bash
pip install -r requirements.txt
```

#### 5. 运行实验

**步骤 1：启动控制面板**
在终端执行入口文件：

```bash
python main.py
```

**步骤 2：配置实验参数**
在弹出的 UI 面板中填入核心信息（Subject ID、模式、范式、物理参数及串口路径）。
_注：开启 **Debug Mode** 后，渲染器将以窗口化（1200x600）启动，并解除屏幕刷新率同步锁，便于单屏设备调试。_

**步骤 3：执行与监控**
点击“开始实验”。若在 Manual 模式，请根据状态栏提示在目标监视器按 `Space` 触发；若在 Auto 模式，系统将自动切入闭环流程。您可通过“数字孪生”模块实时监控进度与硬件反馈。

#### 6. 数据产出规范

实验启动后，系统会在 `data/` 目录下自动生成双轨记录文件。通过 `global_trial_id` 与时间戳实现严格的帧级对齐：

1. `{Subject}_session_{n}_events.csv`：记录低频控制流事件（包含变长 JSON 结构细节）。
2. `{Subject}_session_{n}_kinematics.csv`：记录极高频闭环遥测数据（dx, dy, dz 等）。

#### 7. 框架扩展：添加新范式

无需修改底层的渲染或控制流代码。只需：

1. 在 `src/models/paradigm.py` 中新建类并继承 `BaseParadigm`。
2. 实现 `process_frame()` 等接口：在内存中完成坐标计算，返回标准化的指令字典（包含 `id`, `type`, `pos`, `radius`/`size`）。系统底层将自动接管后续流程。
